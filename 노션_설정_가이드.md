# 노션 설정 가이드
## 사대보험 환급 컨설팅 신청 관리 데이터베이스

---

## STEP 1. 노션 Integration(API 키) 만들기

1. 브라우저에서 [https://www.notion.so/my-integrations](https://www.notion.so/my-integrations) 접속
2. **+ New integration** 클릭
3. 이름 입력 (예: `환급 신청 관리`)
4. 연결할 워크스페이스 선택
5. **Submit** 클릭
6. 생성된 **Internal Integration Secret** 복사 → `.env` 파일의 `NOTION_API_KEY` 에 붙여넣기

---

## STEP 2. 노션 데이터베이스 만들기

1. 노션에서 새 페이지 생성 (빈 페이지)
2. `/database` 입력 → **Table - Full page** 선택
3. 아래 표에 따라 컬럼(속성)을 추가하세요

### 필요한 데이터베이스 컬럼 목록

| 컬럼명 | 속성 타입 | 설명 |
|--------|-----------|------|
| **회사명** | Title (기본값) | 신청 기업명 |
| **상시근로자수** | Number | 직원 수 |
| **담당자 이름** | Text | 담당자 성함 |
| **담당자 연락처** | Phone | 연락처 |
| **담당자 이메일** | Email | 이메일 |
| **퇴직연금 가입 여부** | Select | 가입 / 미가입 / 모름 |
| **퇴직연금 가입 시기** | Date | 가입 시기 |
| **등기임원 사대보험 가입** | Select | 가입 / 미가입 / 모름 |
| **가입자 성명 및 직책** | Text | 성명/직책 목록 |
| **신청일시** | Date | 자동 입력됨 |
| **처리 상태** | Select | 신청 접수 / 진단 중 / 완료 / 불가 |

> ⚠️ **컬럼명은 정확히 위 표와 동일하게** 입력해야 합니다. 오타가 있으면 서버에서 오류가 발생합니다.

#### Select 옵션 설정
- **퇴직연금 가입 여부**: `가입`, `미가입`, `모름`
- **등기임원 사대보험 가입**: `가입`, `미가입`, `모름`
- **처리 상태**: `신청 접수`, `진단 중`, `환급 완료`, `환급 불가`

---

## STEP 3. Integration을 데이터베이스에 연결하기

1. 만든 데이터베이스 페이지 우측 상단 **⋯ (더보기)** 클릭
2. **Connections** → **Add connections** 클릭
3. STEP 1에서 만든 Integration(`환급 신청 관리`) 검색 후 선택
4. **Confirm** 클릭

---

## STEP 4. 데이터베이스 ID 가져오기

데이터베이스를 브라우저에서 열면 URL이 아래와 같습니다:

```
https://www.notion.so/[워크스페이스]/[DATABASE_ID]?v=[VIEW_ID]
```

`?v=` 앞의 32자리 영숫자(하이픈 포함 가능)가 **Database ID**입니다.

예시:
```
https://www.notion.so/myworkspace/a1b2c3d4e5f6...7890?v=...
                                   ↑ 이 부분이 DATABASE_ID
```

복사 후 `.env` 파일의 `NOTION_DATABASE_ID` 에 붙여넣기

---

## STEP 5. 서버 실행하기

### 환경 설정

```bash
# 1. 패키지 설치
pip install -r requirements.txt

# 2. .env 파일 생성 (.env.example 복사)
cp .env.example .env

# 3. .env 파일 열어서 API 키 3개 입력
#    ANTHROPIC_API_KEY=sk-ant-...
#    NOTION_API_KEY=secret_...
#    NOTION_DATABASE_ID=...
```

### 서버 실행

```bash
# 개발용 (로컬 테스트)
python server.py

# 운영용 (권장)
gunicorn server:app -b 0.0.0.0:5000
```

서버가 실행되면 브라우저에서 `http://서버IP:5000` 으로 접속하면 신청 페이지가 열립니다.

## 외부 공유 방법

### 방법 A. 클라우드 서버에 배포 (권장)
AWS EC2, Google Cloud, NCP(네이버클라우드) 등에 서버를 띄우고 도메인을 연결하면
홈페이지 배너에서 이 URL로 링크하면 됩니다.

### 방법 B. Render.com 무료 배포
1. [render.com](https://render.com) 회원가입
2. **New Web Service** → GitHub 저장소 연결
3. Environment Variables에 `.env` 값들 입력
4. 자동으로 배포 URL 생성 (`https://xxx.onrender.com`)

### 방법 C. 로컬 PC에서 실행 + ngrok 터널
임시 테스트용으로만 권장합니다.
```bash
# ngrok 설치 후
ngrok http 5000
# → 외부 URL 생성됨
```

## 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 노션에 데이터가 안 들어옴 | API 키 또는 DB ID 오류 | `.env` 파일 재확인 |
| 노션에 데이터가 안 들어옴 | Integration 연결 누락 | STEP 3 재확인 |
| 컬럼명 오류 | 컬럼명 오타 | 정확한 한글 컬럼명 사용 |
| 명함 OCR 실패 | Anthropic API 키 오류 | API 키 재확인 |
| CORS 오류 | 도메인 설정 | server.py의 CORS 설정 수정 |
